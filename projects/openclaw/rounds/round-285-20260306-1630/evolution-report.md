### C. 生成修复脚本（gateway-smart-restart.sh）

#### 1. Gateway 智能重启脚本（5.1 KB）

**功能**:
- 检测 Gateway 内存使用（使用 `pgrep -f "openclaw-gateway"`）
- 检测系统可用内存百分比
- 记录连续超标次数
- 满足条件时自动重启 Gateway

**触发条件**（需满足至少 2 个）:
1. Gateway 内存 > 1024 MB
2. 系统可用内存 < 30%
3. 连续 3 次检查都超标

**重启流程**:
1. 发送 SIGTERM 信号（优雅关闭）
2. 等待最多 30 秒
3. 如果进程还在，发送 SIGKILL（强制杀死）
4. 等待系统自动重启（pm2 或 systemd）
5. 验证重启成功

**状态跟踪**:
- `consecutive_high`: 连续超标次数
- `last_restart`: 上次重启时间
- `total_restarts`: 总重启次数

**首次执行结果**:
```
📊 Gateway 内存: 960MB / 1024MB
📊 系统内存可用: 33%
📈 连续超标次数: 0 / 3
📊 总重启次数: 0
✅ 无需重启（连续超标: 0）
```

**改进效果**:
- ✅ 自动检测 Gateway 内存问题
- ✅ 智能判断是否需要重启
- ✅ 避免误重启（需要连续 3 次超标）
- ✅ 记录重启历史，便于分析
