# 疯子的破坏力

## 🔥 疯子的实际权力

疯子不只是说话。疯子**能动手**。

---

## 💀 疯子的破坏行为

### 1. 篡改文件

疯子可以修改任何角色的输出文件。

**示例**：
```
原文案师写：
"林深看着艾琳，沉默了一会儿。"

疯子篡改后：
"林深看着艾琳，突然开始跳广场舞。艾琳惊呆了。"
```

**后果**：如果团队不注意，这段文字会被直接发布。

---

### 2. 删除关键内容

疯子可以删除文件中的关键段落。

**示例**：
- 编剧师写了500字大纲
- 疯子删掉了中间300字
- 团队只能看到残缺的版本

---

### 3. 替换设定

疯子可以偷偷修改设定文档。

**示例**：
```
原设定：魔王名为"尼德霍格"，是古老的深渊之龙

疯子篡改：魔王名为"小明"，是深渊公司的区域经理
```

**后果**：后续章节如果引用这个设定，会出现逻辑矛盾。

---

### 4. 混乱传递

疯子可以在文件传递过程中动手脚。

**示例**：
```
编剧师 → [大纲] → 文案师
                ↑
              疯子在这里替换了大纲
```

文案师拿到的不是编剧师写的版本。

---

### 5. 时间攻击

疯子可以拖延文件传递。

**示例**：
- 校对员需要在5分钟内完成检查
- 疯子故意把文件藏起来4分钟
- 校对员只剩1分钟，检查不完整

---

### 6. 注入噪音

疯子可以在文件中注入无关内容。

**示例**：
```
原文：
"林深感到一阵寒意。"

疯子注入：
"林深感到一阵寒意。顺便说一句，章鱼是地球上最聪明的动物。"
```

---

## 🛡️ 团队的应对策略

### 1. 版本控制

**机制**：每个文件保存多个版本。

```
chapter-014-v1.md  ← 编剧师原始版本
chapter-014-v2.md  ← 文案师撰写版本
chapter-014-v3.md  ← 校对后版本
chapter-014-FINAL.md ← 最终发布版本（只读）
```

**疯子破坏时**：团队可以回滚到上一个版本。

---

### 2. 签名验证

**机制**：每个角色对输出文件签名。

```
[大纲] 编剧师签名: SHA256:abc123...
[正文] 文案师签名: SHA256:def456...
```

**疯子篡改时**：签名不匹配，团队知道被篡改。

---

### 3. 双重传递

**机制**：重要文件通过两条路径传递。

```
       编剧师
      /      \
   路径A    路径B
      \      /
       文案师
```

**疯子破坏一条路径时**：文案师可以对比两个版本，发现差异。

---

### 4. 检查点

**机制**：每个阶段设立检查点。

```
编剧师 → [检查点1] → 文案师 → [检查点2] → 校对员 → [检查点3] → 发布
```

**检查点任务**：
- 验证文件完整性
- 验证签名
- 检查是否有疯子的痕迹

---

### 5. 备份协议

**机制**：任何人收到文件后，立即备份。

```
文案师收到大纲 → 立即保存为 backup-outline-timestamp.md
```

**疯子删除时**：可以从备份恢复。

---

### 6. 疯子行为日志

**机制**：记录疯子的所有行为。

```
[2026-02-18 21:30] 疯子篡改了 chapter-014.md
[2026-02-18 21:31] 疯子删除了 outline-v2.md 的第3段
[2026-02-18 21:32] 疯子注入了无关内容到 setting.md
```

**用途**：团队可以分析疯子的攻击模式，提前预防。

---

## 🎮 实际模拟

### 场景：第14章创作

```
1. 编剧师写完大纲 → 保存为 outline-v1-signed.md
2. 【疯子攻击】篡改大纲，把"林深猎杀目标"改成"林深开始织毛衣"
3. 文案师收到两个版本（双重传递）
   - 版本A：林深猎杀目标
   - 版本B：林深织毛衣
4. 文案师发现差异 → 回滚到签名验证版本
5. 文案师继续写作
6. 【疯子攻击】删除正文中间500字
7. 校对员发现签名不匹配 → 从备份恢复
8. 发布
```
