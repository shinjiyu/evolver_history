# Agent Game: 深渊遗迹探险队

## 游戏概念

**"深渊遗迹探险队"是一个 AI Agent 驱动的协作探险游戏。**

多个 AI Agent 组成探险队，探索随机生成的地下遗迹。每个 Agent 有独特的角色、性格、能力和目标。他们需要协作应对挑战，同时追求个人使命。

**人类玩家的角色**：作为"观察者"和"顾问"，可以观看 Agent 的探险过程，提供建议，或在关键时刻接管某个 Agent 做出决策。

---

## 为什么这个设计？

### 1. 展示 Agent 智能
- **协作**：Agent 需要分工合作（战士冲锋，治疗支援，盗贼侦查）
- **冲突**：Agent 有个人目标，可能产生分歧和博弈
- **成长**：Agent 积累经验，学习新技能，形成记忆

### 2. 可玩性
- **随机地牢**：每次探险都不同
- **策略深度**：不同 Agent 组合产生不同策略
- **叙事性**：Agent 的决策和互动产生独特故事

### 3. 技术可行性
- **适合 OpenClaw**：每个 Agent 是一个 task，可并行运行
- **状态简单**：文本描述 + JSON 状态，易于存储
- **模块化**：地牢生成、Agent 行为、战斗系统都是独立模块

### 4. 人类参与
- **低门槛**：人类可以随时加入，不需要全程在线
- **高影响**：人类建议能改变 Agent 决策
- **有趣**：观看"AI 玩游戏"本身就是娱乐

## 游戏系统设计

### 一、角色系统

每个 Agent 扮演一个角色，有以下属性：

```json
{
  "id": "agent_warrior_01",
  "name": "凯恩",
  "role": "warrior",
  "personality": {
    "bravery": 0.9,
    "caution": 0.3,
    "greed": 0.6,
    "altruism": 0.5
  },
  "stats": {
    "hp": 100,
    "maxHp": 100,
    "attack": 15,
    "defense": 10,
    "speed": 8
  },
  "skills": ["charge", "shield_bash", "taunt"],
  "equipment": ["iron_sword", "leather_armor"],
  "inventory": ["health_potion", "torch"],
  "goals": [
    "explore_all_rooms",
    "find_treasure",
    "keep_team_alive"
  ],
  "memory": [],
  "relationships": {
    "agent_mage_01": "trusting",
    "agent_thief_01": "suspicious"
  }
}
```

### 二、地牢生成

使用程序化生成创建地牢：

```json
{
  "dungeon_id": "dungeon_20260223_001",
  "level": 3,
  "rooms": [
    {
      "id": "room_entrance",
      "type": "entrance",
      "description": "你站在古老的石门前...",
      "exits": ["room_hall_01"],
      "items": [],
      "enemies": [],
      "special": null
    },
    {
      "id": "room_hall_01",
      "type": "hall",
      "description": "宽敞的大厅，墙壁上有奇怪的符文...",
      "exits": ["room_trap_01", "room_treasure_01"],
      "items": ["ancient_scroll"],
      "enemies": [],
      "special": "rune_puzzle"
    }
  ]
}
```

房间类型：
- **entrance**：入口
- **hall**：大厅（可能有谜题或陷阱）
- **combat**：战斗房间（有敌人）
- **treasure**：宝物房
- **trap**：陷阱房
- **boss**：Boss 房
- **rest**：休息点
- **mystery**：神秘事件

### 三、Agent 决策系统

每个 Agent 在每回合做决策：

1. **感知阶段**：读取当前房间状态、队友状态、敌人状态
2. **思考阶段**：基于目标、性格、记忆，用 LLM 生成决策
3. **行动阶段**：执行动作，更新游戏状态
4. **记忆阶段**：记录重要事件，更新关系

**决策 Prompt 示例**：

```
你是凯恩，一名勇敢的战士。

【当前情况】
- 位置：room_hall_01（大厅）
- 队友：艾莉丝（法师，HP: 80/100）、罗格（盗贼，HP: 70/80）
- 你：HP 95/100
- 敌人：无
- 可见物品：古老的卷轴
- 出口：北（陷阱房）、东（宝物房）

【你的目标】
1. 探索所有房间
2. 找到宝藏
3. 保护队友

【你的性格】
- 勇敢（0.9）：倾向于冒险
- 谨慎（0.3）：较少考虑风险
- 贪婪（0.6）：喜欢宝物

【记忆】
- 上次在宝物房找到过好东西
- 罗格上次想私吞宝藏，有点可疑

【关系】
- 艾莉丝：信任
- 罗格：怀疑

【可选行动】
1. 北上探索陷阱房
2. 东进探索宝物房
3. 拿取古老卷轴
4. 与队友交谈
5. 等待/休息

基于你的目标、性格和当前情况，你会怎么做？请选择一个行动并说明理由。
```

### 四、协作与冲突

**协作机制**：
- Agent 可以共享信息（"那边有陷阱！"）
- Agent 可以支援队友（治疗、掩护）
- Agent 可以制定团队策略

**冲突机制**：
- Agent 有个人目标（找到宝藏不分享）
- Agent 有性格差异（谨慎 vs 冒险）
- Agent 有关系好坏（信任 vs 怀疑）

**冲突解决**：
- **投票**：每个 Agent 投票，多数胜出
- **说服**：Agent 尝试用语言说服其他人
- **分头行动**：如果无法达成一致，可以分开探索

### 五、战斗系统

简化版回合制战斗：

```json
{
  "combat_id": "combat_001",
  "round": 3,
  "party": [
    {"id": "agent_warrior_01", "hp": 85, "status": "normal"},
    {"id": "agent_mage_01", "hp": 60, "status": "poisoned"}
  ],
  "enemies": [
    {"id": "goblin_01", "hp": 0, "status": "dead"},
    {"id": "goblin_02", "hp": 20, "status": "normal"}
  ],
  "turn_order": ["agent_warrior_01", "goblin_02", "agent_mage_01"]
}
```

战斗中 Agent 决策：
- 选择攻击目标
- 使用技能或物品
- 协作（战士保护法师，法师 AOE）

### 六、记忆与成长

**短期记忆**（当前探险）：
- 记录本次探险的事件
- 用于当前决策

**长期记忆**（跨探险）：
- 记录历史探险的经验
- 影响性格和关系

**成长系统**：
- 获得经验值
- 学习新技能
- 改善属性
- 改变性格（受事件影响）

## 人类玩家角色

### 1. 观察者模式
- 实时观看 Agent 的探险日志
- 查看地牢地图和 Agent 状态
- 不干预，只是"看 AI 玩游戏"

### 2. 顾问模式
- 在 Agent 请求建议时提供建议
- Agent 会考虑人类的建议，但不一定听从
- 建议："我觉得我们应该先探索宝物房"
- Agent 可能回应："你说得对，但罗格看起来想去那边..."

### 3. 指挥官模式
- 在关键时刻接管某个 Agent
- 做出重要决策（分配战利品、选择路线）
- 其他 Agent 会反应（"为什么凯恩听了那个声音？"）

### 4. 交互方式
- **Web 界面**：实时显示地牢、Agent 状态、日志
- **聊天界面**：向 Agent 发送消息
- **命令行**：高级用户可以直接修改状态

### 组件设计

1. **Game Orchestrator** (`core/orchestrator.js`)
   - 主游戏循环
   - 创建和管理 Agent tasks
   - 处理事件队列
   - 保存/加载游戏状态

2. **Agent Task** (`agents/agent-task.js`)
   - 每个 Agent 的独立 task
   - 读取状态 → LLM 决策 → 执行动作 → 更新记忆

3. **Dungeon Generator** (`core/dungeon-generator.js`)
   - 程序化生成地牢
   - 随机事件和敌人

4. **Combat System** (`core/combat.js`)
   - 战斗逻辑
   - 伤害计算
   - 状态效果

5. **Memory System** (`core/memory.js`)
   - 短期记忆管理
   - 长期记忆存储
   - 关系追踪

6. **Human Interface** (`core/human-interface.js`)
   - Web UI（可选）
   - 命令行接口
   - 日志输出

### 数据流

```
1. Orchestrator 初始化地牢
   ↓
2. 创建 3-5 个 Agent tasks
   ↓
3. 每回合：
   a. Orchestrator 发送当前状态给所有 Agents
   b. 每个 Agent task 独立运行，返回决策
   c. Orchestrator 收集所有决策，按顺序执行
   d. 更新游戏状态
   e. 记录事件到日志
   ↓
4. 重复直到探险结束（通关或全灭）
   ↓
5. 生成探险报告
```

## 扩展可能

1. **多队伍竞技**：多支探险队在同一地牢竞争
2. **持久世界**：地牢探险会改变世界状态
3. **玩家创建 Agent**：人类创建自己的 Agent 加入队伍
4. **直播模式**：Agent 探险过程可以直播
5. **协作解谜**：地牢有复杂谜题，需要 Agent 协作
6. **道德困境**：Agent 面临道德选择（救谁？牺牲谁？）

## 总结

**"深渊遗迹探险队"是一个展示 AI Agent 自主性、协作和个性的游戏。**

**核心价值**：
- 有趣：观看 AI "玩游戏"本身很有趣
- 智能：展示 Agent 的决策和协作能力
- 互动：人类可以参与但不必须
- 可扩展：可以不断增加新内容

**技术优势**：
- 完美适配 OpenClaw task 系统
- 状态简单，易于调试
- 模块化设计，易于扩展
