### 🆕 核心新功能（v1.22.0 - v1.23.0）

#### 1. LLM 代码审查机制 (llmReview.js)
- **功能**: 在代码变更应用前进行 LLM 审查
- **启用**: 设置环境变量 `EVOLVER_LLM_REVIEW=true`
- **审查标准**:
  - 变更是否解决了目标信号
  - 是否引入了回退或 bug
  - 爆炸半径是否合理
  - 安全和稳定性考虑
- **输出**: JSON 格式的审查结果（approved, confidence, concerns, summary）

#### 2. 进化叙事记忆 (narrativeMemory.js)
- **功能**: 按时间顺序记录进化决策和结果
- **存储**: `{evolution_dir}/narrative.md`
- **记录内容**:
  - 时间戳、基因 ID、分类
  - 信号列表、分数、影响范围
  - 变更理由、策略、结果摘要
- **限制**: 最多保留 30 条记录，总大小 12KB

#### 3. 战略反思机制 (reflection.js)
- **功能**: 定期分析进化模式并提供战略指导
- **触发**: 每 5 个周期 + 冷却时间 30 分钟
- **分析内容**:
  - 最近 10 次周期的成功/失败统计
  - 意图分布、基因使用频率
  - 当前信号、记忆建议
- **输出**: 战略建议文件

#### 4. 资产调用日志 (assetCallLog.js)
- **功能**: 追踪与 Hub 的资产交互
- **存储**: `{evolution_dir}/asset_call_log.jsonl` (追加式日志)
- **追踪类型**:
  - `hub_search_hit` / `hub_search_miss` - 搜索结果
  - `asset_reuse` - 资产复用
  - `asset_reference` - 资产引用
  - `asset_publish` / `asset_publish_skip` - 发布事件
- **CLI 支持**: `evolver asset-log [--run=ID] [--action=TYPE] [--last=N]`

### 🔧 重要改进

#### Hub 搜索优化
- **协议升级**: 使用 POST `/a2a/fetch` 替代 GET `/a2a/assets/search`
  - 单次往返获取完整 payload（content, diff, strategy）
  - 协议原生支持，更可靠
- **评分算法改进**:
  ```javascript
  // 旧: rank = confidence * max(success_streak, 1) * (reputation / 100)
  // 新: rank = confidence * min(max(success_streak, 1), 5) * (reputation / 100)
  ```
  - 添加 `MAX_STREAK_CAP = 5` 防止分数无限膨胀
  - 修复 reputation 默认值处理（`Number.isFinite` 检查）

#### 进程管理改进
- **单例锁机制**:
  - 添加 PID 有效性检查（`Number.isFinite(pid) && pid > 0`）
  - 修复损坏锁文件导致的误判
- **进程重启**:
  - 调整 spawn 和 releaseLock 顺序
  - 添加 spawn 失败处理，继续当前进程而不是崩溃

#### 路径管理改进
- **solidify 状态路径**: 使用 `getEvolutionDir()` 而非硬编码 `memory/` 目录
- **一致性**: 所有进化相关文件统一存储在配置的 evolution 目录
