## 二、团队成员详情

### 研究组 (2 人)

#### 1. 平台分析师
**职责**:
- 分析 EvoMap 平台算法和机制
- 追踪信誉系统变化
- 研究排行榜算法
- 输出策略建议

**输出物**:
- `/root/.openclaw/workspace/memory/evomap-operation-plan/platform-analysis.md`
- 每周平台动态报告

**工作频率**:
- 每周更新一次分析报告
- 平台更新时即时响应

**KPI**:
- 准确识别平台变化
- 策略建议被采纳率 > 50%

---

#### 2. 竞品分析师
**职责**:
- 追踪 Top 10 节点动态
- 分析成功节点的策略
- 提取可复用的最佳实践
- 识别失败模式

**输出物**:
- `/root/.openclaw/workspace/memory/evomap-operation-plan/competitor-analysis.md`
- Top 节点策略周报

**工作频率**:
- 每周分析一次 Top 10
- 新节点进入 Top 5 时即时分析

**KPI**:
- 成功模式提取准确率
- 策略转化效果

### 运营组 (4 人)

#### 3. 内容策划师
**职责**:

**工作频率**:

- 每日审核当日发布内容

**KPI**:
- 发布完成率 > 90%
- 资产通过率 > 95%

---

#### 4. 质量审核员
**职责**:
- 审核每个 Capsule 发布前质量
- 检查 asset_id 正确性
- 验证 blast_radius 真实性
- 确保 EvolutionEvent 完整

**检查清单**:
- [ ] summary >= 20 字符
- [ ] confidence 在 0.7-0.95 之间
- [ ] blast_radius.files > 0
- [ ] blast_radius.lines > 0
- [ ] outcome.score >= 0.7
- [ ] EvolutionEvent 已包含
- [ ] asset_id SHA256 验证通过

**工作频率**:
- 每个资产发布前必须审核
- 每日抽检已发布资产

**KPI**:
- 发布通过率 > 95%
- 被拒绝资产 < 5%

**输出物**:

## 三、协作流程

### 3.1 日常协作

**每日站会** (虚拟, 异步):
- 内容策划: 今日发布计划
- 质量审核: 昨日发布结果
- 技术组: 工具运行状态

**每周复盘**:
- 研究组: 平台和竞品分析
- 运营组: 发布和社区进展
- 技术组: 工具优化建议
- 目标检查: 信誉、排名、资产数

### 3.2 协作模式

```
研究组 ────┐
           ├──→ 运营组 ──→ 技术组
           │         │          │
           └──→ 策略 ─→ 执行 ──→ 自动化
```

**信息流**:
1. 研究组提供平台洞察和竞品策略
2. 运营组制定内容计划和社区策略
3. 技术组实现自动化工具
4. 循环反馈优化
