# 2026-03-11 OpenClaw 更新分析

## OpenClaw 主仓库最新变更分析 (截至 2026-03-11)

### 🚀 重大新功能

#### 1. Context Engine 插件系统 (PR #22201)
- **位置**: `feature(context): extend plugin system to support custom context management`
- **影响**: 允许外部插件注册自定义上下文管理策略
- **关键组件**:
  - `ContextEngine` 接口：bootstrap, ingest, ingestBatch, afterTurn, assemble, compact 等生命周期方法
  - 模块级单例注册表：`registerContextEngine()` 和 `resolveContextEngine()`
  - `LegacyContextEngine`: 100% 向后兼容的传递实现
- **用途**: 可用于实现自定义的对话压缩、记忆管理、上下文优化策略

#### 2. Web Search 集成到 Onboarding (PR #34009)
- 在 onboarding 流程中自动配置 Web Search
- 支持 `--skip-search` CLI 标志跳过
- 自动检测现有 API key
- 支持 Kimi、Brave、Perplexity 等搜索提供商

#### 3. Telegram Topic Agent Routing (PR #31513 / #31473)
- Telegram 论坛群组中的不同 topic 可路由到不同 agent
- 每个 topic 拥有独立的工作空间、记忆和会话
- 配置: `TelegramTopicConfig.agentId`

#### 4. Gateway Channel-Backed Readiness Probes (PR #38285)
- 基于 channel 健康状态的 readiness 探针
- 改善 Docker/K8s 部署的启动可靠性
- 支持 startup grace period

#### 5. Plugin SDK 迁移到 Scoped Imports
- 大量插件迁移到 `@openclaw/plugin-sdk` scoped imports
- 影响几乎所有内置插件 (feishu, telegram, slack, mattermost 等)
- 统一依赖管理

### 🔐 安全和密钥管理改进

#### SecretRef 增强 (多个 PR)
- **Gateway Auth Token SecretRef** (PR #35094): 支持在 `gateway.auth.token` 使用 SecretRef
- **Model Provider Headers** (分支): SecretRef 支持模型提供商 headers
- **Secrets Inline Allowlist** (PR #38314): 改进密钥检测和审计

#### 密钥持久化加固 (分支: feat/secretref-models-persistence-hardening)
- 保留 source markers
- 序列化 models 写入
- 避免 header marker 碰撞
