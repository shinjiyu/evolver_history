## P0 问题（严重）

### 1. 高频 API Rate Limit (429)

**问题描述**: GLM-5 模型频繁触发速率限制

**统计数据**:
- 总错误数: 91 次
- 平均错误率: 15.2 次/小时
- 峰值时段: 08:00（38 次错误）
- 最大连续错误: 44 次

**时间分布**:
| 时段 | 错误数 | 占比 |
|------|--------|------|
| 08:00 | 38 | 41.8% |
| 09:00 | 17 | 18.7% |
| 07:00 | 16 | 17.6% |
| 03:00 | 11 | 12.1% |
| 其他 | 9 | 9.8% |

**错误示例**:
```
429 该模型当前访问量过大，请您稍后再试
429 您的账户已达到速率限制，请您控制请求频率
```

**根本原因分析**:
1. 多个 cron 任务在相近时间触发（07:00-09:00 是高峰期）
2. 缺乏请求速率控制和退避机制
3. 并行任务过多导致瞬时请求量过大

**改进建议**:
1. **错峰调度**: 调整 cron 任务时间，避免 07:00-09:00 时段重叠
2. **实现退避机制**: 
   ```javascript
   // 指数退避重试
   const retryWithBackoff = async (fn, maxRetries = 3) => {
     for (let i = 0; i < maxRetries; i++) {
       try {
         return await fn();
       } catch (e) {
         if (e.status === 429) {
           const delay = Math.pow(2, i) * 1000; // 1s, 2s, 4s
           await sleep(delay);
         } else throw e;
       }
     }
   };
   ```
3. **请求队列**: 实现请求队列，限制并发数
