**严重程度**: 🔴 P0 - 紧急
**出现次数**: 76 次
**影响范围**: 所有长时间运行的会话

**详细描述**:
对话历史超过了模型的上下文窗口限制，导致会话中断，任务失败。

**示例**:
```json

  "timestamp": "2026-03-09T20:01:00.000Z"
}
```

**根本原因**:
1. 长时间运行的会话累积了大量历史消息
2. MEMORY.md 文件过大（4.7 MB），每次加载都占用大量上下文
3. 工具调用结果包含大量数据（如文件内容）
4. 没有上下文窗口管理机制

**影响**:
- 会话中断，需要重新开始
- 长时间任务无法完成
- 用户体验差

**建议修复方案**:
1. **短期**:
   - 实现 MEMORY.md 精炼（已完成，从 4.7 MB → 5.7 KB）
   - 添加上下文窗口监控
   - 当接近限制时，自动压缩历史消息

2. **中期**:
   - 实现增量式记忆加载（只加载相关部分）
   - 添加对话历史摘要机制
   - 限制单次工具调用返回的数据量

3. **长期**:
   - 实现上下文窗口管理策略
   - 智能选择保留哪些历史消息
   - 实现分层的记忆系统
