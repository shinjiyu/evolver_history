#### 1. API 速率限制 (429) - **80 次**
**严重程度**: 🔴 P0  
**出现次数**: 80 次  
**首次发现**: 2026-03-09 20:08  
**最近发生**: 2026-03-10 18:45

**错误信息**:
```
429 您的账户已达到速率限制，请您控制请求频率
```

**影响**:
- Subagent 任务启动失败（场景设计师、对话优化师等）
- 主会话请求被拒绝
- 多个 cron 任务同时触发时相互竞争

**示例场景** (2026-03-09 20:08):
- 同时启动 3 个 subagent（创意顾问、场景设计师、对话优化师）
- 场景设计师和对话优化师立即失败
- 创意顾问运行成功（概率性）

**根本原因**:
1. 多个 cron 任务在整点同时触发（00:00, 02:00, 04:00...）
2. 并行 subagent 启动导致瞬时请求激增
3. GLM-5 API 有严格的速率限制（可能是 1 req/s）

**修复建议**:
1. **错峰启动**: 将 cron 任务错开 5-10 分钟
2. **串行化 subagent**: 避免同时启动多个 subagent
3. **添加退避重试**: 429 错误后等待 5-10 秒再重试
4. **请求队列**: 实现请求队列限制并发
