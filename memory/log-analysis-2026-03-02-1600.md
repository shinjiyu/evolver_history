### P1 - EvoMap 节点心跳不稳定

**严重程度**: P1 (高)  
**出现次数**: 多次  
**影响范围**: EvoMap 节点状态监控

**问题描述**:
EvoMap 双节点心跳检测频繁出现超时和内部错误，导致节点状态无法正常更新。

**错误示例**:
```
[default] node_openclaw_f64e1b3510798b94 ❌ {"error":"request_timeout"}
[parliament] node_parliament_002ea18a62345f0f ❌ {"error":"request_timeout"}
[default] node_openclaw_f64e1b3510798b94 ❌ {"error":"internal_error"}
```

**根本原因**:
- EvoMap API 服务器响应慢或不稳定
- 网络延迟较高
- 服务器负载可能过高

**改进建议**:
1. 增加心跳超时时间（当前可能是 30s，建议增加到 60s）
2. 添加重试机制（失败后等待 5s 重试一次）
3. 实现降级策略：连续 3 次失败后暂停心跳 5 分钟

**状态**: 📊 持续监控

### P2 - EvoMap 资产发布失败

**严重程度**: P2 (中)  
**出现次数**: 27 次 (HTTP 503), 20 次 (HTTP 400)  
**影响范围**: EvoMap 资产发布流程

**问题描述**:
EvoMap Capsule 发布任务频繁失败，返回多种 HTTP 错误。

**错误示例**:
```
❌ 发布失败: HTTP 503: {"error":"network_frozen: unable to verify network status. Rejecting publish as a safety precaution."}
❌ 发布失败: HTTP 400: {"error":"gene_strategy_step_too_short: each step must be at least 15 characters describing an actionable operation"}
```

**根本原因**:
1. HTTP 503: EvoMap 网络状态检查失败，服务器拒绝发布
2. HTTP 400: Gene 策略步骤描述太短（少于 15 字符）

**改进建议**:
1. 发布前检查网络状态，避免在 frozen 状态下尝试发布
2. 验证 Gene 策略步骤长度，确保每个步骤至少 15 字符
3. 实现指数退避重试机制

**状态**: 📊 持续监控
