## 🔍 发现的问题

### P0 - 子代理并发启动导致 429 限流激增（严重）

**症状**:
```
时间段: 12:07-12:15, 16:07-16:10
错误次数: 12+ 次
失败率: 67% (2/3 子代理失败)
```

**典型场景**:
```
16:07:40 - 启动 3 个子代理（文字编辑、节奏分析师、人物塑造顾问）
16:08:30 - 立即触发 429 错误
16:08:38 - 第二次 429 错误
16:08:47 - 第三次 429 错误
结果: 2/3 子代理失败
```

**时间分布**:
| 场景 | 子代理数 | 429 错误 | 失败率 |
|------|----------|----------|--------|
| 小说评审（常规）| 3 | 9 次 | 67% |
| 小说评审（灵感）| 3 | 5+ 次 | 67% |
| 一致性检查 | 2-3 | 5+ 次 | 50%+ |

**影响**: 高 - 导致大量任务失败，系统效率低下

**根因分析**:
1. 子代理并发启动时同时发送 API 请求
2. 智谱 AI API 对并发请求敏感
3. 高峰期（16:00-20:00）API 压力大
4. 缺少子代理启动间隔控制

**改进**: 创建 evolved-subagent-stagger Skill，实现智能调度

## 🔧 实施的改进

### A. 创建新 Skill

#### 1. evolved-subagent-stagger

**目的**: 智能调度子代理启动，避免并发触发 API Rate Limit

**核心功能**:
- **串行化启动**: 任务有序执行，完全避免 429
- **延迟启动**: 错开 API 请求时间，最终并行执行
- **自适应调度**: 根据 API 状态自动调整延迟

**推荐策略**:
| 场景 | 延迟时间 | 说明 |
|------|----------|------|
| 正常时段 | 5 秒 | API 压力较小 |
| 高峰期（16:00-20:00）| 10 秒 | API 压力大 |
| 429 错误频繁 | 15 秒 | 限流严重 |

**文件**:
- `skills/evolved-subagent-stagger/SKILL.md` (4563 bytes)
- `evolver/fixes/subagent-stagger.sh` (4574 bytes)

**使用示例**:
```bash
# 显示当前状态和推荐配置
bash /root/.openclaw/workspace/evolver/fixes/subagent-stagger.sh --status

# 生成 3 个子代理的启动计划
bash /root/.openclaw/workspace/evolver/fixes/subagent-stagger.sh --plan 3

# 显示推荐延迟
bash /root/.openclaw/workspace/evolver/fixes/subagent-stagger.sh --delay
```
