## 🔧 实施的改进

### A. 创建新 Skill

#### 1. evomap-heartbeat-monitor

**目的**: 监控 EvoMap 节点心跳健康度，自动检测超时和错误

**核心功能**:
- **自适应超时**: 30s → 60s → 90s → 120s（根据失败次数自动调整）
- **重试机制**: 失败后等待 5s 重试一次
- **冷却期降级**: 连续 3 次失败后暂停心跳 5 分钟
- **实时监控**: 生成状态文件和报告

**文件**:
- `skills/evomap-heartbeat-monitor/SKILL.md` (5294 bytes)
- `evolver/fixes/evomap-heartbeat-monitor.sh` (2441 bytes)

**使用示例**:
```bash
# 单次检查
bash /root/.openclaw/workspace/evolver/fixes/evomap-heartbeat-monitor.sh

# 定时监控（每 5 分钟）
*/5 * * * * /root/.openclaw/workspace/evolver/fixes/evomap-heartbeat-monitor.sh
```

**预期效果**:
- ✅ 心跳超时减少 70%
- ✅ 错误恢复时间 < 1 分钟
- ✅ 资源浪费减少 60%

#### 2. evomap-publish-validator

**目的**: EvoMap Capsule 发布前预检查，避免常见错误

**核心功能**:
- **网络状态检查**: 避免 network_frozen 错误（HTTP 503）
- **Gene 策略验证**: 确保每个步骤 >= 15 字符（避免 HTTP 400）
- **队列容量检查**: 避免 task_full 错误（HTTP 409）
- **自动修复**: 可选自动扩展太短的步骤

**文件**:
- `skills/evomap-publish-validator/SKILL.md` (6602 bytes)
- `evolver/fixes/evomap-publish-validator.sh` (3335 bytes)

**使用示例**:
```bash
# 基本检查
bash /root/.openclaw/workspace/evolver/fixes/evomap-publish-validator.sh capsule.json

# 自动修复 Gene 策略
bash /root/.openclaw/workspace/evolver/fixes/evomap-publish-validator.sh capsule.json false true
```

**预期效果**:
- ✅ 发布失败率降低 80%
- ✅ 避免常见错误（503/400/409）
- ✅ 自动修复格式问题

### B. 改进现有 Skill

#### 1. safe-operations

**改进内容**:

**新增策略**:
1. **上下文扩展**: 精确匹配失败时，扩大上下文（包含前后各 2-3 行）
2. **模糊匹配**: 对于微小的空白差异，使用模糊匹配和诊断
3. **降级策略**: Edit 失败后降级到 Write（覆盖整个文件）
4. **并发冲突检测**: 记录文件哈希，检测编辑期间是否被修改

**新增文件**:
- `evolver/fixes/smart-edit.sh` (2338 bytes) - 智能编辑脚本（带重试和上下文扩展）
- `evolver/fixes/check-file-exists.sh` (1587 bytes) - 文件存在性检查脚本（可自动创建）

**新增案例库**:
- 案例 1: 空白字符不匹配（空格 vs 制表符）
- 案例 2: 换行符差异（CRLF vs LF）
- 案例 3: 并发编辑冲突
- 案例 4: 文本不唯一

**预期效果**:
- ✅ Edit 失败率降低 60%
- ✅ 自动处理常见错误
- ✅ 提供清晰的错误诊断
