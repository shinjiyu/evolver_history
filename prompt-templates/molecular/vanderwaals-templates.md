# 范德华键模板（Van der Waals Templates）

> **目标**：激活多路径探索，扩展推理空间
> 
> **适用场景**：开放性问题、需要创新的问题、存在多种解法的任务

---

## 模板 V1: 双路径对比（Dual Path Comparison）

### 设计原理

同时探索两条推理路径，通过对比验证答案。

### 模板内容

```
【目标】同时探索两条不同的推理路径，对比结果

【指令】
对于此问题，请同时构建两条独立的推理路径：

【路径 A】（常规/直接方法）
├─ 步骤A1: ...
├─ 步骤A2: ...
└─ 结论A: ...

【路径 B】（替代/逆向方法）
├─ 步骤B1: ...
├─ 步骤B2: ...
└─ 结论B: ...

【对比分析】
├─ 结论一致性: [A和B是否得到相同结果]
├─ 复杂度对比: [哪条路径更简洁]
├─ 可靠性对比: [哪条路径更不易出错]
└─ 最终选择: [选择哪条路径的结论，为什么]

如果两条路径结论不同，分析差异原因并解决。

【示例】
问题：求 1+2+3+...+100 的和。

【路径 A】（直接求和）
├─ 步骤A1: 这是一个等差数列
├─ 步骤A2: 使用求和公式 S = n(a₁+aₙ)/2
├─ 步骤A3: S = 100(1+100)/2 = 5050
└─ 结论A: 5050

【路径 B】（配对法）
├─ 步骤B1: 将数列首尾配对：(1,100), (2,99), ...
├─ 步骤B2: 每对之和为101
├─ 步骤B3: 共50对，总和 = 50 × 101 = 5050
└─ 结论B: 5050

【对比分析】
├─ 结论一致性: ✅ 完全一致
├─ 复杂度对比: 路径A需要记住公式，路径B更直观
├─ 可靠性对比: 都很可靠
└─ 最终选择: 两者都正确，确认答案为5050

现在请解决以下问题：
{question}
```

### 使用说明

- **适用问题类型**：有多种解法的数学问题、逻辑问题
- **预期行为分布**：Exploration ~40%, Deep Reasoning ~40%
- **注意事项**：可能增加推理时间，但提高可靠性

## 模板 V2: 多假设验证（Multi-Hypothesis Validation）

### 设计原理

生成多个假设，逐一验证，选择最优。

### 模板内容

```
【目标】同时考虑多种可能的假设，逐一验证

【指令】
首先列出所有可能的解题方向（假设），然后逐一验证：

【假设生成】
H1: [假设1描述]
H2: [假设2描述]
H3: [假设3描述]
...

【假设验证】
对于每个假设：
├─ H1
│   ├─ 推理过程: ...
│   ├─ 验证结果: [成立/部分成立/不成立]
│   └─ 置信度: [高/中/低]
├─ H2
│   └─ ...
...

【假设整合】
├─ 可排除的假设: [列表]
├─ 需要进一步验证的假设: [列表]
└─ 最可能的假设: [选择]

基于验证结果，给出最终答案。

【示例】
问题：小明参加数学竞赛得了前三名，他可能是第几名？

【假设生成】
H1: 小明是第一名
H2: 小明是第二名
H3: 小明是第三名

【假设验证】
├─ H1: 小明是第一名
│   ├─ 推理过程: 符合"前三名"条件
│   ├─ 验证结果: 可能成立
│   └─ 置信度: 中（无更多信息）
├─ H2: 小明是第二名
│   ├─ 推理过程: 符合"前三名"条件
│   ├─ 验证结果: 可能成立
│   └─ 置信度: 中
└─ H3: 小明是第三名
    ├─ 推理过程: 符合"前三名"条件
    ├─ 验证结果: 可能成立
    └─ 置信度: 中

【假设整合】
├─ 可排除的假设: 无（所有都符合条件）
├─ 需要进一步验证的假设: H1, H2, H3
└─ 最可能的假设: 无法确定，需要更多信息

最终答案：小明可能是第一名、第二名或第三名中的任何一个。

现在请解决以下问题：
{question}
```

### 使用说明

- **适用问题类型**：不确定性问题、推理题、诊断问题
- **预期行为分布**：Exploration ~50%, Deep Reasoning ~30%
- **注意事项**：需要合理限制假设数量

## 模板 V3: 竞争性推理（Competitive Reasoning）

### 设计原理

让不同推理策略"竞争"，选出最优策略的结果。

### 模板内容

```
【目标】让不同推理策略"竞争"，选出最优

【指令】
采用"辩论赛"模式，让三种策略竞争：

【策略A：快速直觉】
- 依赖第一反应和常识
- 快速但可能不准确
- 时间限制：3步内

【策略B：系统分析】
- 逐步分解，详细推理
- 慢但准确
- 无步骤限制

【策略C：反例驱动】
- 先假设结论，再找反例
- 通过否定来逼近真理
- 迭代式

【竞争过程】
Round 1: 各策略给出初步答案
├─ 策略A: [答案A]
├─ 策略B: [答案B]
└─ 策略C: [答案C]

Round 2: 各策略攻击其他策略的弱点
├─ A攻击B: ...
├─ B攻击C: ...
└─ C攻击A: ...

Round 3: 各策略修正自己的答案
├─ 策略A修正: ...
├─ 策略B修正: ...
└─ 策略C修正: ...

【裁判判决】
基于准确性、一致性、鲁棒性，选择获胜策略的答案。

【示例】
问题：一个房间里有3盏灯，门外有3个开关，每个开关控制一盏灯。
你只能进房间一次，如何确定每个开关控制哪盏灯？

Round 1: 各策略给出初步答案
├─ 策略A: 打开一个开关，进去看哪盏灯亮。
│   └─ 答案: 无法确定所有对应关系
├─ 策略B: 先开开关1等5分钟，关掉，开开关2，进去。
│   └─ 答案: 灯亮的是开关2，灯热的是开关1，剩下一个是开关3
└─ 策略C: 这个问题无解吗？
    └─ 答案: 等等，B的方法似乎可行

Round 2: 各策略攻击其他策略
├─ A攻击B: B利用了灯泡发热的特性，这是否合理？
│   └─ B回应: 题目没说是什么灯，传统白炽灯会发热
├─ B攻击C: C没有给出解法
│   └─ C回应: 我承认B的解法更完整
└─ C攻击A: A只进一次，但只能确定一个对应关系
    └─ A回应: 确实，我的方法不完整

Round 3: 各策略修正
├─ 策略A修正: B的方法更优，我放弃
├─ 策略B修正: 坚持我的方法，但需说明假设灯泡会发热
└─ 策略C修正: 采用B的方法

裁判判决: 策略B获胜
├─ 准确性: 高（假设灯泡发热）
├─ 一致性: 高
└─ 鲁棒性: 中（依赖于灯泡类型假设）

现在请解决以下问题：
{question}
```

### 使用说明

- **适用问题类型**：开放性问题、策略选择问题
- **预期行为分布**：Exploration ~45%, Deep Reasoning ~35%
- **注意事项**：输出较长，适合重要问题

## 模板 V4: 思维风暴（Brainstorming）

### 设计原理

先生成大量可能的思路，再筛选和整合。

### 模板内容

```
【目标】通过思维风暴生成多个可能的解题方向

【指令】
【Phase 1: 发散】（5分钟思维风暴）
不考虑可行性，列出所有可能的相关想法：
├─ 想法1: ...
├─ 想法2: ...
├─ 想法3: ...
...（至少列出5个）

【Phase 2: 聚合】
对每个想法进行评估：
├─ 想法1: [可行性: 高/中/低] [相关性: 高/中/低]
├─ 想法2: ...
...

【Phase 3: 整合】
选择最有希望的2-3个想法：
├─ 选择1: ... (原因: ...)
├─ 选择2: ... (原因: ...)
└─ 选择3: ... (原因: ...)

【Phase 4: 执行】
基于选择的想法，构建完整解决方案。

【示例】
问题：如何提高程序的运行效率？

【Phase 1: 发散】
├─ 想法1: 使用更快的算法
├─ 想法2: 并行化处理
├─ 想法3: 缓存结果
├─ 想法4: 减少IO操作
├─ 想法5: 使用更高效的数据结构
├─ 想法6: 编译优化
├─ 想法7: 减少内存分配
├─ 想法8: 使用GPU加速

【Phase 2: 聚合】
├─ 想法1: [可行性: 高] [相关性: 取决于问题]
├─ 想法2: [可行性: 中] [相关性: 高]
├─ 想法3: [可行性: 高] [相关性: 高]
├─ 想法4: [可行性: 高] [相关性: 高]
├─ 想法5: [可行性: 高] [相关性: 高]
├─ 想法6: [可行性: 低] [相关性: 中]
├─ 想法7: [可行性: 高] [相关性: 中]
├─ 想法8: [可行性: 低] [相关性: 取决于问题]

【Phase 3: 整合】
├─ 选择1: 想法3 - 缓存 (最通用)
├─ 选择2: 想法5 - 数据结构 (基础优化)
└─ 选择3: 想法4 - 减少IO (常见瓶颈)

【Phase 4: 执行】
基于这3个方向，建议的优化策略：
1. 首先分析程序热点，找出最耗时的部分
2. 对频繁访问的数据使用缓存
3. 选择合适的数据结构（如HashMap代替List查找）
4. 批量处理IO，减少系统调用次数

现在请解决以下问题：
{question}
```

### 使用说明

- **适用问题类型**：设计问题、优化问题、开放性问题
- **预期行为分布**：Exploration ~60%, Deep Reasoning ~25%
- **注意事项**：适合初始探索阶段，不适合有明确最优解的问题
