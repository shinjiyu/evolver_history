# 组合模板（Combined Templates）

> **目标**：整合多种"键"的优势，构建稳定的分子结构
> 
> **适用场景**：复杂问题、需要全面考虑的任务

---

## 模板 C+H: 共价键+氢键（强逻辑+反思）

### 设计原理

将严密的逻辑链与主动反思结合，在推进的同时不断验证。

### 模板内容

```
【目标】构建严密的逻辑链，并在关键节点进行反思验证

【指令】
采用"推进-检查"循环模式：

【Phase 1: 推进】（构建2-3步逻辑链）
→ 步骤N: [基于前序步骤的逻辑推理]
→ 步骤N+1: ...
→ 步骤N+2: ...

【Phase 2: 检查】（对上述步骤进行反思）
→ 检查点:
   ├─ 逻辑连贯性: [步骤间是否合理衔接]
   ├─ 前提可靠性: [是否依赖未经证实的前提]
   └─ 结论合理性: [中间结论是否合理]

【Phase 3: 决策】
→ 如果检查通过 → 继续 Phase 1
→ 如果发现问题 → 回退并修正

循环直到得到最终答案。

【格式示例】
[推进1] 因为A，所以B
[推进2] 因为B，所以C
[检查1] 验证 B→C 的逻辑
   └─ 结论: 通过
[推进3] 因为C，所以D
[检查2] 验证 C→D 的逻辑
   └─ 结论: 发现问题 - C的前提不完整
[回退] 回到推进2，重新分析...

现在请解决以下问题：
{question}
```

### 预期行为分布

- Deep Reasoning: ~50%
- Self-Reflection: ~30%
- Self-Exploration: ~10%

### 适用问题类型

- 数学证明
- 逻辑推理
- 程序设计
- 需要高准确性的任务

## 模板 C+V: 共价键+范德华键（强逻辑+探索）

### 设计原理

在保持逻辑主干的同时，探索替代路径。

### 模板内容

```
【目标】在保持逻辑严密性的同时，探索多种可能的推理路径

【指令】
采用"主干-分支"结构：

【主干构建】（Deep Reasoning）
首先构建最直接的推理主线：
主线步骤1 → 主线步骤2 → ... → 主线结论

【分支探索】（Exploration）
在关键决策点，同时探索替代路径：
决策点N:
├─ 主线路径: [继续原定方向]
├─ 分支路径A: [探索替代方向1]
└─ 分支路径B: [探索替代方向2]

【路径收敛】
├─ 如果分支路径更优 → 切换到分支作为新主线
├─ 如果主线路径更优 → 继续主线
└─ 如果需要综合 → 整合多条路径

【注意】
- 主线必须始终保持逻辑连贯
- 分支探索不应该破坏主线的完整性
- 及时放弃无效分支，避免资源浪费

【示例】
问题：证明 √2 是无理数。

【主干构建】
主线: 假设 √2 = p/q（最简分数），然后推导矛盾

【分支探索】
在"如何证明 p 和 q 不互质"这个决策点：
├─ 主线路径: 证明 p² = 2q² ⇒ p 是偶数
├─ 分支路径A: 使用无限下降法
└─ 分支路径B: 利用素因子分解

【路径收敛】
主线路径（偶数论证）最直接，继续使用：
1. p² = 2q² ⇒ p 是偶数
2. 设 p = 2m ⇒ 4m² = 2q² ⇒ q² = 2m²
3. q 也是偶数
4. 与 p、q 互质矛盾

现在请解决以下问题：
{question}
```

### 预期行为分布

- Deep Reasoning: ~55%
- Self-Reflection: ~15%
- Self-Exploration: ~30%

### 适用问题类型

- 数学问题（有多种解法）
- 算法设计
- 问题求解

## 模板 H+V: 氢键+范德华键（反思+探索）

### 设计原理

通过反思和探索不断优化推理方向，适合不确定性高的问题。

### 模板内容

```
【目标】通过反思和探索，不断优化推理方向

【指令】
采用"假设-验证-调整"循环：

【Step 1: 假设生成】（Exploration）
生成多个可能的解题方向：
├─ 方向A: ...
├─ 方向B: ...
└─ 方向C: ...

【Step 2: 快速验证】（Reflection）
对每个方向进行快速可行性检查：
├─ 方向A: [可行/不可行，原因]
├─ 方向B: [可行/不可行，原因]
└─ 方向C: [可行/不可行，原因]

【Step 3: 方向选择】
选择最有希望的方向，深入推理

【Step 4: 深度反思】
对选定方向的推理过程进行深度审查：
├─ 逻辑漏洞扫描
├─ 边界条件测试
└─ 一致性检查

【Step 5: 迭代调整】
如果发现问题：
├─ 小问题 → 修正当前方向
└─ 大问题 → 回到Step 1，选择其他方向

循环直到找到可靠答案。

【示例】
问题：设计一个算法，找出数组中出现次数超过一半的元素。

【Step 1: 假设生成】
├─ 方向A: 使用哈希表计数
├─ 方向B: 排序后取中间元素
└─ 方向C: Boyer-Moore 投票算法

【Step 2: 快速验证】
├─ 方向A: [可行] 简单直接，但需要O(n)空间
├─ 方向B: [可行] 但排序需要O(nlogn)时间
└─ 方向C: [可行] O(n)时间，O(1)空间

【Step 3: 方向选择】
选择方向C（投票算法），因为最优

【Step 4: 深度反思】
├─ 逻辑漏洞: 算法正确性是否保证？
│   └─ 答: 是的，超过一半的元素一定会胜出
├─ 边界条件: 空数组？只有一个元素？
│   └─ 答: 需要特殊处理
└─ 一致性检查: 是否真的不需要额外空间？
    └─ 答: 只需要两个变量（candidate和count）

【Step 5: 迭代调整】
需要添加边界条件处理和最终验证。

现在请解决以下问题：
{question}
```

### 预期行为分布

- Deep Reasoning: ~30%
- Self-Reflection: ~35%
- Self-Exploration: ~35%

### 适用问题类型

- 开放性问题
- 设计问题
- 不确定性高的任务

## 模板 C+H+V: 完整分子结构（三种键的整合）

### 设计原理

完整复现论文中的分子结构理论，包含三种化学键的协同作用。

### 模板内容

```
【目标】构建具有稳定分子结构的完整推理过程

【指令】
将推理过程建模为分子结构，包含三种化学键：

【结构规划】
根据问题类型，规划键分布：
├─ Deep Reasoning (共价键): ~60% - 主干推理
├─ Self-Reflection (氢键): ~20% - 关键节点验证
└─ Self-Exploration (范德华键): ~20% - 分支探索

【执行框架】

1. 【初始化】建立推理起点
   → 明确问题、定义变量、列出已知条件

2. 【主干扩展】（Deep Reasoning）
   → 构建连续的逻辑链
   → 格式：[D-N] 因为 [依据], 所以 [结论]
   → 保持：每步引用前序步骤

3. 【关键节点验证】（Self-Reflection）
   → 在重要结论处暂停
   → 执行反审计
   → 格式：[R-N] 验证 [D-M]...
   → 如果发现问题，回退到相应步骤

4. 【分支探索】（Self-Exploration）
   → 在不确定性高的节点
   → 同时探索多个方向
   → 格式：[E-N] 替代路径: ...
   → 比较并选择最优路径

5. 【收敛整合】
   → 主线推理收敛到答案
   → 验证最终答案
   → 给出置信度评估

【键分布监控】
在推理过程中，保持对键分布的监控：
├─ 如果 Deep Reasoning < 40% → 加强逻辑连贯性
├─ 如果 Self-Reflection < 10% → 增加验证节点
└─ 如果 Self-Exploration > 30% → 减少分支，聚焦主线

【结构稳定性检查】
最终答案给出前，检查：
□ 逻辑折叠是否完成（主线是否收敛）
□ 是否存在未解决的分支
□ 反思节点是否覆盖所有关键决策

【示例】
问题：找出数组中第K大的元素。

【初始化】
问题：在数组arr中找出第K大的元素
已知：数组长度n，目标排名K
目标：返回第K大的元素

【主干扩展】
[D-1] 最直接的方法是排序，然后取第(n-K)个元素
[D-2] 排序时间复杂度是O(nlogn)

【分支探索】
[E-1] 替代路径：使用快速选择算法（Quick Select）
    → 平均时间复杂度O(n)，最坏O(n²)
[E-2] 替代路径：使用最小堆维护K个最大元素
    → 时间复杂度O(nlogK)

【关键节点验证】
[R-1] 验证 [D-1]：排序后取第(n-K)个是否正确？
    → 检查：是的，升序排列后，第(n-K)个就是第K大
[R-2] 比较 [D-2]、[E-1]、[E-2] 的时间复杂度
    → 快速选择平均最优，但堆方法更稳定

【收敛整合】
选择方案：根据需求选择
├─ 如果K较小 → 使用最小堆
├─ 如果需要稳定性能 → 使用堆
└─ 如果追求平均最优 → 使用快速选择

最终答案：提供两种方案及其适用场景。

现在请解决以下问题：
{question}
```

### 预期行为分布

- Deep Reasoning: ~60%
- Self-Reflection: ~20%
- Self-Exploration: ~20%

### 适用问题类型

- 复杂问题
- 需要全面考虑的任务
- 重要决策问题

## 快速选择指南

| 问题类型 | 推荐模板 | 原因 |
|---------|---------|------|
| 数学计算 | C+H | 需要严密逻辑+验证 |
| 数学证明 | C+H 或 C+H+V | 逻辑严密性最重要 |
| 逻辑推理 | C+H | 逻辑+验证 |
| 程序设计 | H+V 或 C+H+V | 需要探索+验证 |
| 算法问题 | C+H+V | 需要全面考虑 |
| 开放性问题 | H+V 或 C+H+V | 需要探索和反思 |
| 快速任务 | C+H | 平衡速度和质量 |
| 重要决策 | C+H+V | 需要全面分析 |

## 模板自定义指南

### 调整键分布

可以通过调整指令中的比例来改变模型的推理行为：

```
【调整为更保守】（减少探索，增加验证）
├─ Deep Reasoning: ~50%
├─ Self-Reflection: ~35%
└─ Self-Exploration: ~15%

【调整为更探索】（增加探索，减少验证）
├─ Deep Reasoning: ~45%
├─ Self-Reflection: ~15%
└─ Self-Exploration: ~40%

【调整为更严密】（增加逻辑，减少其他）
├─ Deep Reasoning: ~70%
├─ Self-Reflection: ~15%
└─ Self-Exploration: ~15%
```

### 根据模型能力调整

- **强推理模型**：可以使用完整的C+H+V模板
- **一般模型**：推荐使用C+H模板
- **快速响应需求**：使用简化的C+H模板
