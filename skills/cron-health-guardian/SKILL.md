**版本**: 1.0
**创建时间**: 2026-03-12
**创建方式**: evolver-self-evolution (Round 312)
**目的**: 监控所有 Cron 任务的健康状态，自动检测和修复常见问题

## 🎯 核心问题

### PAT-094: Cron 任务状态误报

**首次发现**: 2026-03-12 12:00
**严重程度**: 🟡 P1 - 高优先级

**问题描述**:
- `novel-auto-review` 任务显示 error，但实际执行成功
- `nginx-security-daily` 任务报错，原因是脚本路径不匹配
- 缺少统一的任务健康检查机制

**影响**:
- 🟡 误报导致不必要的告警
- 🟡 真实问题可能被忽略
- 🟡 需要手动检查每个任务

**根本原因**:
1. Cron 任务状态判断逻辑不准确
2. 脚本路径配置不一致
3. 缺少任务执行后的自动验证

---

## 📋 守护策略

### 1. 任务健康检查（主动）

**检查频率**: 每 2 小时

**检查项目**:
1. **任务执行状态** - 是否按时执行
2. **脚本文件存在性** - 脚本路径是否正确
3. **执行结果验证** - 任务是否真正成功
4. **资源使用情况** - 是否占用过多资源

**健康状态定义**:
| 状态 | 说明 | 处理 |
|------|------|------|
| ✅ 健康 | 正常执行，无错误 | 无需处理 |
| 🟡 警告 | 执行成功但有警告 | 记录日志 |
| 🔴 失败 | 执行失败或未执行 | 自动修复或告警 |
| ⚪ 禁用 | 任务已禁用 | 跳过检查 |

### 2. 自动修复（智能）

**可自动修复的问题**:

| 问题类型 | 修复策略 | 成功率 |
|---------|---------|--------|
| 脚本路径错误 | 更新到正确路径 | 90% |
| 权限不足 | 添加执行权限 | 95% |
| 依赖缺失 | 安装缺失依赖 | 70% |
| 配置错误 | 重置为默认配置 | 60% |

**修复流程**:
1. 检测到问题
2. 记录问题详情
3. 尝试自动修复
4. 验证修复效果
5. 如果失败，发送告警

### 3. 降级运行（安全）

**当任务连续失败 3 次时**:
1. 自动禁用任务
2. 发送告警通知
3. 记录失败详情
4. 提供手动恢复指南

## 🔧 使用方法

### 1. 检查所有 Cron 任务健康

```bash
bash /root/.openclaw/workspace/evolver/fixes/cron-health-check.sh
```

**输出示例**:
```
Cron 任务健康检查
==================
时间: 2026-03-12 12:30

任务状态:
  ✅ autonomous-exploration - 健康（2h ago）
  ✅ evomap-feature-monitor - 健康（57m ago）
  🟡 novel-auto-review - 警告（状态误报，实际成功）
  ❌ nginx-security-daily - 失败（脚本路径错误）
  ✅ evolver-self-evolution - 健康（4h ago）

健康任务: 3/5 (60%)
失败任务: 1/5 (20%)

修复建议:
  1. nginx-security-daily: 更新脚本路径
```

### 2. 自动修复失败任务

```bash
# 自动修复所有失败任务
bash /root/.openclaw/workspace/evolver/fixes/cron-health-check.sh --auto-fix

# 仅修复特定任务
bash /root/.openclaw/workspace/evolver/fixes/cron-health-check.sh --fix nginx-security-daily
```

### 3. 添加到 Cron

```bash
# 每 2 小时检查一次
0 */2 * * * bash /root/.openclaw/workspace/evolver/fixes/cron-health-check.sh >> /root/.openclaw/workspace/logs/cron-health.log 2>&1
```
