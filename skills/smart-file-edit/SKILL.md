# Smart File Edit - 智能文件编辑策略

解决 Edit 工具精确匹配失败问题（PAT-032），提高文件编辑成功率。

## 核心问题

**PAT-032 / PAT-046**: Edit 工具无法找到目标文本
- 出现次数: 28+ 次（Round 252-258，持续增加）
- 根本原因:
  1. 文件内容被修改，编辑脚本使用旧文本
  2. 空格/换行符不匹配
  3. 文件内容与预期不一致
  4. **新问题（2026-03-01）**: 小说章节等频繁更新的文件更易失败

## 编辑策略

### 策略 1: 先读后编（推荐）

**黄金法则**: 编辑前总是先读取最新内容

```markdown
## 错误做法
直接使用旧文本进行编辑

## 正确做法
1. 先 read 文件
2. 从读取的内容中精确复制要替换的文本
3. 使用 read 的内容进行 edit
```

### 策略 2: 使用 write 替代 edit

当编辑内容较多时，使用 write 更安全：

```markdown
## 适用场景
- 大段内容更新
- 文件结构变化
- 多处修改

## 步骤
1. read 文件完整内容
2. 在内存中修改
3. write 整个文件
```

### 策略 3: 分段编辑

将大编辑分解为多个小编辑：

```markdown
## 问题
大段文本匹配困难

## 解决
1. 识别关键行
2. 分多次小编辑
3. 每次只改 1-2 行
```

### 策略 4: 容错匹配

使用更灵活的匹配策略：

```markdown
## 原始文本（可能失败）
"  - item1\n  - item2\n  - item3"

## 容错策略
1. 只匹配关键部分
2. 忽略尾部换行符差异
3. 使用更短的 oldText
```

## 最佳实践

### ✅ 推荐做法

1. **编辑前读取**
   ```
   Step 1: read file.md
   Step 2: 从读取结果中复制精确文本
   Step 3: edit file.md (使用复制的文本)
   ```

2. **使用唯一标识**
   ```
   # 选择包含唯一标识的行
   oldText: "| PAT-032 | [操作] Edit 工具匹配失败"
   # 而不是
   oldText: "| 操作 |"  # 太通用，可能多次匹配
   ```

3. **验证编辑结果**
   ```
   edit file.md
   read file.md  # 验证修改是否成功
   ```

### ❌ 避免做法

1. **不要假设文件内容**
   ```
   # 错误：假设文件内容没变
   edit file.md (使用昨天读取的内容)
   ```

2. **不要忽略空格差异**
   ```
   # 错误：空格数量不匹配
   oldText: "- item"  # 实际是 "  - item"
   ```

3. **不要编辑只读文件**
   ```
   # 错误：evolver_history 是 git 仓库
   edit /root/.openclaw/workspace/evolver_history/...
   ```

## 自动化改进

### 检测脚本

```bash
#!/bin/bash
# 检测文件是否被修改

FILE=$1
LAST_MODIFIED=$(stat -c %Y "$FILE" 2>/dev/null)
NOW=$(date +%s)
AGE=$((NOW - LAST_MODIFIED))

if [[ $AGE -lt 300 ]]; then
  echo "⚠️ 文件在 $AGE 秒前被修改，建议重新读取"
fi
```

### 编辑前检查

```markdown
## 检查清单
- [ ] 文件路径正确
- [ ] 文件存在且可写
- [ ] 最近没有其他进程修改
- [ ] oldText 来自最新读取
- [ ] newText 格式正确
```

## 错误恢复

当 edit 失败时：

1. **立即读取文件**
   ```
   read file.md
   ```

2. **分析差异**
   ```
   对比预期内容 vs 实际内容
   ```

3. **选择替代方案**
   ```
   - 使用 write 完全重写
   - 使用更精确的 oldText
   - 分解为多个小编辑
   ```

4. **记录经验**
   ```
   更新 SKILL.md 记录失败原因
   ```
