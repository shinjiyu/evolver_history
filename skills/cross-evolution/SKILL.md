# 交叉进化框架 (Cross-Evolution Framework)

通过**交叉分析**解决自指悖论，通过**进化历史追踪**避免循环修复。

## 核心问题

Agent 自我改进面临两个根本问题：

1. **自指悖论**：一个系统无法完整地分析和改进自身（哥德尔不完备）
2. **局部循环**：没有时间维度的记忆，会在同一组问题上反复发现、反复修复

## 解决方案

```
        Round 1 (交叉归纳)
    ┌───────────────────────┐
    │  归纳1 ──分析──► 进化   │
    │  归纳2 ──分析──► 归纳   │
    └───────────────────────┘
              ↓
        Round 2 (交叉进化)
    ┌───────────────────────┐
    │  进化1 ──改进──► 归纳   │
    │  进化2 ──改进──► 进化   │
    └───────────────────────┘
```

**关键原则**：
- 归纳不分析自己
- 进化不改进自己
- 历史独立存储，Append-only

## 子技能

| 技能 | 用途 | 触发条件 |
|------|------|----------|
| `log-analysis` | 分析日志，识别问题模式 | "分析日志"、"归纳问题" |
| `skill-mining` | 从模式提炼 Skill/Rule | "生成 skill"、"提炼模式" |
| `evolution-history` | 追踪进化历史，检测循环 | 自动调用 |
| `neutral-evaluator` | 中立评估，避免被引导 | "评估"、"评价" |

## 快速开始

### 1. 中立评估（推荐首选）

当需要评估某个项目/库/观点时：

```
用户：评估 https://github.com/xxx/yyy
```

触发 `neutral-evaluator`，输出结构化评估报告。

### 2. 日志分析 → Skill 生成

```
用户：分析我的会话日志，提炼可复用的 Skill
```

1. 读取 `~/.openclaw/agents/main/sessions/*.jsonl`
2. 识别重复模式和错误
3. 输出结构化报告
4. 生成 Skill 文件

### 3. 完整交叉进化

```
用户：启动交叉进化
```

执行两轮分析（需要多个子 agent 并行）。

## OpenClaw 适配说明

| 原始 (Cursor) | OpenClaw |
|---------------|----------|
| `.cursor/rules/` | 合并到 SKILL.md |
| `.cursor/agents/` | 用 sessions_spawn 或 skill 触发 |
| `agent-transcripts/*.txt` | `~/.openclaw/agents/main/sessions/*.jsonl` |
| `.cursor/evolution-store.json` | `workspace/evolution-store.json` |

## 防止被引导的机制

本框架特别强调**独立判断**：

1. **显式记录判断** - 每次评估后固化结论
2. **矛盾检测** - 后续对话矛盾时提示
3. **先验证后判断** - 强制检查证据（如"有没有可执行代码"）
4. **置信度标注** - 明确说明判断的可信程度
