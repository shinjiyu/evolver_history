# 日志分析技能

分析 OpenClaw 会话日志，识别问题模式和可复用工作流。

## 数据源

OpenClaw 的会话日志位于：
```
~/.openclaw/agents/main/sessions/*.jsonl
```

每个文件是一个会话的完整记录。

## 分析流程

### Step 1: 读取日志

```bash
# 列出最近活跃的会话
ls -lt ~/.openclaw/agents/main/sessions/*.jsonl | head -10

# 读取特定会话
read ~/.openclaw/agents/main/sessions/<session-id>.jsonl
```

### Step 2: 提取关键信息

从每条消息中提取：

| 字段 | 来源 | 说明 |
|------|------|------|
| 角色 | `message.role` | user / assistant |
| 内容 | `message.content` | 文本或工具调用 |
| 工具调用 | `content[].toolCall` | 使用的工具 |
| 错误 | `errorMessage` | LLM 返回的错误 |
| 结果 | `tool_result` | 工具执行结果 |

### Step 3: 识别模式

**错误模式**：
- 工具调用失败
- LLM 输出格式错误
- 超时/中断

**重复工作流**：
- 相同的命令序列出现多次
- 相似的文件操作模式
- 固定的工具组合

**机会信号**：
- 用户反复请求同一功能
- 用户表达不满
- 用户提出改进建议

### Step 4: 输出报告

```markdown
# 日志分析报告

## 元数据
- 分析日期：...
- 日志数量：N 个会话
- 时间范围：...

## 概要
| 指标 | 数值 |
|------|------|
| 错误数 | N |
| 重复工作流 | N |
| 改进建议 | N |

## 问题列表

### P1: [问题标题]
- **症状**：...
- **根因**：...
- **证据**：session-id:行号
- **建议**：...

### P2: ...

## 重复工作流

### W1: [工作流名称]
- **步骤**：
  1. ...
  2. ...
- **出现次数**：N
- **建议**：可提炼为 Skill

## 分析局限
- 未覆盖：...
- 需验证：...
```

## 历史追踪

如果存在 `evolution-history/` 目录：

1. 读取 `pattern-registry.md`
2. 将发现与已有模式比对
3. 标注「复现 PAT-XXX」或「新发现」
4. 避免重复报告已知问题

## 证据引用规范

- 格式：`session-id:行号` 或 `session-id:起始-结束`
- 禁止：只有文件名没有行号
- 禁止：模糊表述如"大约"、"可能"

## 严重性分级

| 等级 | 定义 | 说明 |
|------|------|------|
| P | 严重 | 必须修复 |
| W | 警告 | 应该修复 |
| S | 建议 | 可以改进 |
