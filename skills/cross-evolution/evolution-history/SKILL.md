# 进化历史追踪

记录演进历史，检测循环修复，避免重复劳动。

## 存储结构

```
workspace/evolution-history/
├── manifest.md           # 总览
├── pattern-registry.md   # 模式注册表
├── rounds/               # 轮次记录
│   ├── round-001.md
│   └── round-002.md
└── archive/              # 归档
```

## Manifest 模板

```markdown
# 进化总览

## 元信息
| 字段 | 值 |
|------|-----|
| 首轮日期 | ... |
| 最近轮次 | Round N |
| 总轮次 | N |
| 活跃模式数 | N |

## 最近轮次
| 轮次 | 日期 | 新发现 | 已解决 | 复现 |
|------|------|--------|--------|------|

## 收敛趋势
| 指标 | 趋势 |
|------|------|
| 新发现速率 | 下降/稳定/上升 |
| 复现率 | ... |
```

## Pattern Registry 模板

```markdown
# 模式注册表

| ID | 指纹 | 类型 | 首现 | 末现 | 次数 | 状态 |
|----|------|------|------|------|------|------|
| PAT-001 | [描述] | error | Round 1 | Round 3 | 3 | 活跃 |
```

### 状态生命周期

```
新发现 → 活跃 → 已缓释 → 已解决 → 归档
                    ↑          │
                    └── 复现 ←─┘
```

## 循环检测

### 检测规则

| 信号 | 含义 | 动作 |
|------|------|------|
| 模式出现 ≥3 次 | 修复不彻底 | 升级为 P0 |
| 连续 2 轮矛盾修改 | 振荡 | 冻结，人工介入 |
| 连续 3 轮无变化 | 已收敛 | 暂停进化 |

### 使用方法

**每轮开始时**：
1. 读取 `manifest.md` 和 `pattern-registry.md`
2. 将本轮发现与已有模式比对
3. 标注「复现 PAT-XXX」或「新发现」

**每轮结束时**：
1. 分配 Pattern ID
2. 更新注册表
3. 创建 Round Record
4. 执行归档（保留最近 5 轮）

## Round Record 模板

```markdown
# Round N

## 日期
...

## 归纳报告
- 输入文件：...
- 发现问题：N 个

## 进化产物
- 新增 Skill：...
- 更新 Skill：...

## 模式变化
| ID | 状态变化 |
|----|----------|
| PAT-001 | 活跃 → 已缓释 |

## 收敛指标
- 新发现：N
- 复现：N
- 已解决：N
```

## 判断固化

重要判断写入：

```markdown
## [日期] 判断固化

### 评估对象：XXX
- 结论：...
- 置信度：.../5
- 证据：...
- 后续变化需确认修正
```
