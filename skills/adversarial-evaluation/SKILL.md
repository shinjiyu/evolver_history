# 对抗式评估 Skill (Adversarial Evaluation)

**核心理念**：通过正反双方的对抗辩论，获得更全面、客观的评估结论。

## ⚡ 快速模式 (Fast Mode)

默认启用快速模式，30秒内完成评估：

```
证据收集(5s) → 正反方并行(15s) → 裁判判决(10s)
```

快速模式优化：
- 论点数量：2 个（足够形成判断）
- 并行执行：正反方同时分析
- 精简输出：只保留核心结论

## 触发条件

- "对抗评估 XXX" / "正反分析 XXX" / "辩证评估 XXX"
- 需要避免被用户引导的中立判断

## 评估流程

### Phase 1: 证据收集（5秒）

快速收集核心事实：

```markdown
## 评估对象
- 名称：...
- 类型：代码库 / 观点 / 决策 / ...

## 核心事实（3-5条）
1. ...
2. ...
3. ...
```

### Phase 2: 并行对抗（15秒）

**同时**招募正反方 subagent：

| 角色 | 任务 | 输出限制 |
|------|------|----------|
| **支持者** | 2个优点 + 价值主张 | ≤100字 |
| **反对者** | 2个问题 + 风险警告 | ≤100字 |

#### 支持者模板（精简版）

```
你是支持者。快速论证【评估对象】的优点。

输出（≤100字）：
## 正方
1. [优点1] - 证据：...
2. [优点2] - 证据：...
核心价值：[一句话]
```

#### 反对者模板（精简版）

```
你是反对者。快速批判【评估对象】的问题。

输出（≤100字）：
## 反方
1. [问题1] - 影响：...
2. [问题2] - 影响：...
最大风险：[一句话]
```

### Phase 3: 裁判判决（10秒）

裁判快速综合：

```
## ⚖️ 判决

| 维度 | 正方 | 反方 |
|------|------|------|
| 论点强度 | X/5 | Y/5 |

**结论**：[一句话]

**建议**：[适用/不适用，因为...]
```

## 执行命令（快速模式）

```
# 并行执行正反方
Promise.all([
  spawn(支持者, { timeout: 15 }),
  spawn(反对者, { timeout: 15 })
]).then(([pro, con]) => {
  // 立即执行裁判
  spawn(裁判, { context: { pro, con }, timeout: 10 });
});
```

## 输出报告（精简版）

```markdown
# 对抗评估：[对象名]

## 证据
- ...

## 🔵 正方
1. ...
2. ...
核心价值：...

## 🔴 反方
1. ...
2. ...
最大风险：...

## ⚖️ 判决
正方 X/5 vs 反方 Y/5

**结论**：...

**建议**：...
```

## 深度模式 (可选)

当用户说"深度对抗评估"时启用：

- 论点数量：3-5 个
- 执行时间：2-3 分钟
- 详细分析：包含证据核查、场景分析

## 防引导机制

1. **角色固定**：支持者必须找优点，反对者必须找问题
2. **并行独立**：正反方同时分析，互不影响
3. **证据要求**：每个论点必须有证据支撑

## 配置

```json5
{
  adversarialEvaluation: {
    fastMode: true,           // 默认快速模式
    fastMinArguments: 2,      // 快速模式论点数
    deepMinArguments: 3,      // 深度模式论点数
    parallelExecution: true,  // 并行执行
    fastTimeout: 30,          // 快速模式总超时(秒)
    deepTimeout: 180          // 深度模式总超时(秒)
  }
}
```

## 使用方式

| 命令 | 模式 | 耗时 |
|------|------|------|
| "对抗评估 XXX" | 快速 | ~30s |
| "正反分析 XXX" | 快速 | ~30s |
| "深度对抗评估 XXX" | 深度 | ~3min |
| "详细正反分析 XXX" | 深度 | ~3min |
