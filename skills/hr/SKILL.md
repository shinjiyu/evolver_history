# HR Skill - 智能团队招募与协作编排

## 概述

HR 是一个团队编排 Skill，能够：
1. **分析需求** - 理解用户目标，确定需要什么样的团队
2. **招募团队** - 创建合适的 subagent 团队成员
3. **制定流程** - 设计协作模式（串行、并行、对抗、迭代等）
4. **执行协调** - 组织 subagent 协同完成目标
5. **汇总结果** - 整合各 subagent 的产出

## 触发条件

当用户说：
- "组建一个团队来..."
- "招募 subagent 完成..."
- "用多 agent 协作..."
- "帮我组织一个编辑部/研发组/评审团..."
- "HR，我需要..."

## 使用示例

### 示例 1：编辑部写小说
```
用户: HR，我需要写一部科幻小说，5章，每章3000字

HR 响应:
1. 分析需求 → 小说创作项目
2. 招募团队:
   - 设定师: 构建世界观
   - 编剧师: 设计剧情
   - 文案师: 撰写正文
   - 评审团: 3个不同视角的读者
3. 协作流程: 设定→剧情→写作→评审→迭代
4. 执行并汇总结果
```

### 示例 2：对抗式分析
```
用户: HR，组织对抗分析这个项目

HR 响应:
1. 分析需求 → 项目评估
2. 招募团队:
   - 支持者: 正面分析优点
   - 反对者: 批判分析问题
   - 裁判: 综合双方观点
3. 协作流程: 并行分析→对抗辩论→综合判决
4. 执行并输出报告
```

### 示例 3：研究项目
```
用户: HR，组建团队调研微服务架构

HR 响应:
1. 分析需求 → 技术调研
2. 招募团队:
   - 资料收集员: 搜索相关资料
   - 技术分析师: 分析优缺点
   - 案例研究员: 研究实际案例
   - 报告撰写员: 整合输出报告
3. 协作流程: 并行收集→串行分析→汇总报告
4. 执行并交付
```

## 协作模式

### 1. 串行模式 (Pipeline)
```
任务A → 任务B → 任务C → 完成
```
适用：有依赖关系的任务

### 2. 并行模式 (Parallel)
```
任务A ─┐
任务B ─┼→ 汇总 → 完成
任务C ─┘
```
适用：独立可并行的任务

### 3. 对抗模式 (Adversarial)
```
正方 → 观点A ─┐
              ├→ 裁判 → 结论
反方 → 观点B ─┘
```
适用：需要平衡视角的分析

### 4. 迭代模式 (Iterative)
```
执行 → 评审 → 修改 → 评审 → ... → 达标
```
适用：需要持续改进的任务

### 5. 分层模式 (Hierarchical)
```
        主管
       / | \
     A1 A2 A3
     |  |  |
    ... ... ...
```
适用：复杂分层任务

## 执行流程

```
1. 接收用户请求
   ↓
2. 分析需求类型和复杂度
   ↓
3. 确定需要的角色和数量
   ↓
4. 设计协作流程
   ↓
5. 使用 sessions_spawn 创建 subagent
   ↓
6. 分配任务给各 subagent
   ↓
7. 监控执行进度
   ↓
8. 收集结果
   ↓
9. 汇总输出
   ↓
10. 向用户报告
```

## 配置选项

```json5
{
  hr: {
    max_team_size: 10,           // 最大团队规模
    default_timeout: 300,         // 默认超时(秒)
    iteration_threshold: 8.0,     // 迭代达标阈值
    max_iterations: 3,            // 最大迭代次数
    subagent_model: "minimax/MiniMax-M2.1"  // subagent 模型
  }
}
```

## 注意事项

1. **成本控制**: 为 subagent 选择合适的模型
2. **超时管理**: 设置合理的任务超时
3. **结果验证**: 检查 subagent 输出质量
4. **错误处理**: 处理 subagent 失败情况
5. **上下文隔离**: 利用 subagent 的天然隔离特性
