## 🎯 核心问题

### PAT-114: 定时任务缺乏统一健康监控（高优先级）

**症状**:
```
问题任务:
- evomap-feature-monitor: error (API 404)
- novel-auto-review: error (Rate limit)
- nginx-security-daily: error (Feishu 配置)
- analyze-openclaw-updates: error (检查中)

持续时间: > 40 小时
发现方式: 手动检查日志
```

**影响**:
- 🔴 **任务失败长期未发现**: 错误持续 40+ 小时才被人工发现
- 🔴 **缺乏自动告警**: 任务失败后无自动通知
- 🔴 **影响系统可靠性**: 关键任务失败影响系统功能
- 🟡 **运维效率低**: 需要手动检查每个任务状态

**根本原因**:
1. **缺少统一的任务健康监控**
2. **没有自动告警机制**
3. **任务失败后缺少自动恢复**
4. **缺少任务执行趋势分析**

## 📋 监控策略

### 1. 分级监控架构

```
┌─────────────────────────────────────────────────┐
│           定时任务健康监控系统                    │
├─────────────────────────────────────────────────┤
│                                                 │
│  Level 3: 趋势分析（长期）                       │
│  └─ 执行成功率趋势 + 异常检测                    │
│                                                 │
│  Level 2: 状态监控（中期）                       │
│  └─ 最近执行状态 + 失败告警                      │
│                                                 │
│  Level 1: 心跳检测（短期）                       │
│  └─ 任务是否运行 + 执行频率                      │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 2. 任务健康评分

**评分维度**:
1. **执行频率** (30%): 是否按预期频率执行
2. **成功率** (40%): 最近 10 次执行的成功率
3. **执行时长** (20%): 是否在正常范围内
4. **错误严重度** (10%): 错误的严重程度

**健康等级**:
- ✅ **健康** (90-100 分): 正常运行
- 🟡 **警告** (70-89 分): 需要关注
- 🔴 **异常** (50-69 分): 需要修复
- ❌ **故障** (< 50 分): 立即处理

## 🚨 告警机制

### 自动告警条件

1. **任务连续失败 3 次**
   - 发送 Feishu 告警
   - 记录到告警日志
   - 标记任务为异常

2. **任务超过预期时间未执行**
   - 发送 Feishu 告警
   - 检查 cron 配置
   - 尝试自动恢复

3. **任务健康评分 < 70**
   - 发送 Feishu 告警
   - 生成诊断报告
   - 建议修复措施

### 告警渠道

1. **Feishu**: 实时告警（需配置）
2. **本地日志**: 所有告警记录
3. **系统通知**: 关键告警
