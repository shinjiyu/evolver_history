# ReCAP 目标分解

## 职责

本 Skill 基于 ReCAP（Recursive Context-Aware Reasoning and Planning）框架，将抽象目标分解为可执行的子任务树。

**核心作用**：将「无限搜索空间」折叠为「任务树上的下一个节点」，使 SCL 的 Cognition 阶段每次只需决策「树上的当前节点」，而不是面对开放性的搜索空间。

---

## 触发条件

- **首次**：TASKS.md 中没有任务树（Agent 刚收到新目标）
- **层级完成**：TASKS.md 的当前层级节点全部标记为 `[x]` 或 `[!]`
- **主动重分解**：scl-memory 建议重分解（连续失败、重要发现改变方向）
- **定期刷新**：距上次分解超过 7 天（环境可能已变化）

---

## 分解流程

### Phase 0：读取上下文

1. 读取 TASKS.md 中的「顶层目标」和「约束条件」
2. 读取 MEMORY.md 中的「领域知识」（已有的知识不重复探索）
3. 读取最近 3 天的 Daily Log（了解已执行内容和当前进展）
4. 若是重分解：读取 TASKS.md 中的已完成节点和失败节点

### Phase 1：现状评估

```
评估以下维度：
- 当前目标的完成度：已完成什么、还差什么
- 当前最大的阻力是什么
- 环境是否发生了影响目标的变化
- 当前最高价值的未完成方向是什么
```

### Phase 2：生成子任务树

**分解原则**：

1. **粒度适中**：每个叶子节点 = 一次心跳可完成的单步行动
2. **优先级排序**：高影响 × 低难度的节点排在前面
3. **依赖明确**：有前置依赖的节点标注（如「1.2 依赖 1.1 完成」）
4. **可验证**：每个节点有明确的完成标准（如「发布 3 篇笔记」而非「发布笔记」）
5. **动态保留**：不要把所有细节都分解到底，保留灵活性给后续重分解

**树的深度建议**：
- 顶层：1-3 个一级子目标
- 二级：每个一级下 2-5 个可执行任务
- 三级（可选）：若某任务复杂，可再分一层

### Phase 3：写入 TASKS.md

```markdown
## 任务树（ReCAP 第 N 轮，YYYY-MM-DD）

**分解依据**：[本次分解的关键假设和当前环境判断]

- [ ] 1. [一级子目标]
  - [ ] 1.1 [具体任务，完成标准：...]
  - [ ] 1.2 [具体任务，完成标准：...]（依赖 1.1）
- [ ] 2. [一级子目标]
  - [ ] 2.1 ...
```

### Phase 4：标记当前执行节点

在写入后，标记下一次心跳应执行的第一个节点为 `[~]`（进行中），并在「执行摘要」表格写入：

```
| YYYY-MM-DD HH:MM | ReCAP 重分解 | 生成 N 个子任务 | 子任务 1.1 |
```

---

## ReCAP 的动态修正机制

ReCAP 不是一次性生成完整计划，而是**前瞻性 + 动态修正**：

- **首轮分解**：基于当前知识生成完整树
- **执行后修正**：每 5 轮或层级完成时，基于实际结果调整剩余节点
- **保留父计划**：执行细节时，始终确保 TASKS.md 的「顶层目标」可见，防止目标漂移

**修正策略**：
```
若子任务 1.2 的实际效果比预期好 → 加大下一个类似任务的投入
若子任务 1.2 连续失败 → 重新分解这个分支，换角度切入
若发现新的重要信息 → 考虑调整整个树的优先级
```

## 抽象目标示例：「运营小红书，三个月涨粉 1000」

```markdown
## 任务树（ReCAP 第 1 轮，2026-03-04）

**分解依据**：
- 当前粉丝数：未知（需先查）
- 假设：仅合规手段，无付费推广预算
- 关键假设：内容质量 > 数量，小红书算法偏好垂直内容

- [ ] 1. 现状调研与策略制定（本周）
  - [~] 1.1 查看当前账号数据（粉丝数、内容表现、赛道）
  - [ ] 1.2 竞品分析（同赛道涨粉快的账号，分析其内容模式）
  - [ ] 1.3 确定内容垂直方向和差异化定位

- [ ] 2. 内容生产与发布（持续）
  - [ ] 2.1 建立选题库（每周 10 个选题，按热度排序）
  - [ ] 2.2 按节奏发布（每周 3-5 篇，测试最优时间段）
  - [ ] 2.3 配图优化（竖图 9:16，高对比度封面）
  - [ ] 2.4 话题标签策略（5-8 个，混合大小流量话题）

- [ ] 3. 互动与增长（持续）
  - [ ] 3.1 回复评论（每日，提升互动率）
  - [ ] 3.2 主动互动同赛道账号（评论、合作）

- [ ] 4. 数据复盘与调整（每周）
  - [ ] 4.1 分析本周笔记数据，识别高表现内容模式
  - [ ] 4.2 根据数据调整策略
```
