# SCL Memory 层

## 职责

本 Skill 实现 SCL R-CCAM 中的 **Memory（记忆）** 阶段，在每次心跳的 Action 完成后执行。

目标：确保本轮所有重要信息都被持久化，使下一次心跳的 Retrieval 阶段能获得准确的当前状态。

---

## 执行流程

### Step 1：写入 Daily Log（每轮必须）

向 `memory/YYYY-MM-DD.md` 追加本轮记录：

```markdown
## HH:MM 心跳

**执行子任务**：[TASKS.md 中的节点编号和名称]
**行动**：[具体执行了什么]
**结果**：[成功/失败/部分完成 + 关键数据/输出摘要]
**发现**：[执行过程中发现的有用信息（如「小红书竖图比横图播放量高 3x」）]
**下轮建议**：[基于本轮结果，下一步最优行动]
```

### Step 2：更新 TASKS.md（每轮必须）

1. 找到本轮执行的子任务节点，更新状态：
   - `[ ]` → `[x]` 若成功完成
   - `[ ]` → `[!]` 若失败（附失败原因）
   - `[~]` → `[x]` 若本轮完成了进行中的任务

2. 在「执行摘要」表格追加本轮行（保留最近 5 行，旧的归档到 Daily Log）：
   ```markdown
   | YYYY-MM-DD HH:MM | 子任务 1.2 | 成功，发布了 3 篇笔记 | 子任务 1.3（互动回复） |
   ```

3. 若发生能力缺口自举，在「已安装 Skill 记录」追加：
   ```markdown
   | YYYY-MM-DD | image-generation | 需要配图生成 | DALL-E API |
   ```

### Step 3：评估是否需要 ReCAP 重分解

检查以下条件，若满足任一，在 Daily Log 标记「建议下轮重分解」：
- 当前层级的所有子任务已完成
- 发现了改变任务方向的重要信息
- 连续 3 次 `[!]` 失败（任务树可能不合理）
- 距上次 ReCAP 已超过 7 天

### Step 4：每 5 轮提炼到 MEMORY.md

每 5 次心跳执行一次（检查 TASKS.md 中的执行摘要行数）：

将 Daily Log 中标记为「发现」的内容整理，更新 MEMORY.md 对应主题：

```markdown
## 领域知识（[日期] 更新）
- 小红书竖图（9:16）比横图播放量高约 3 倍
- 发布时间：晚 7-9 点互动率最高
- 话题标签：5-8 个为最优，超过 10 个反而降权
```

提炼原则：
- 只写**可复用的**通用知识，不写本轮特定的执行细节
- 若与已有知识矛盾，保留新的并标注「待验证」

---

## 写入规范

**Daily Log（memory/YYYY-MM-DD.md）**：
- Append-only，永远不修改已有内容
- 若当天文件不存在，创建并写入标题 `# YYYY-MM-DD`

**TASKS.md**：
- 只更新任务状态和摘要表格
- 不修改任务树结构（结构修改由 ReCAP 负责）
- 若 TASKS.md 不存在（首次运行），创建基础结构

**MEMORY.md**：
- 按主题组织，不破坏已有结构
- 在对应主题下更新或追加
- 若添加新主题，在文件末尾新增章节

---

## 执行完成确认

Memory 阶段执行完毕后，输出简短确认（写入 context）：

```
[SCL-Memory] 本轮记忆写入完成
- Daily Log：✅ 已追加（memory/YYYY-MM-DD.md）
- TASKS.md：✅ 子任务 1.2 标记为 [x]
- 执行摘要：✅ 已更新（第 N 轮）
- MEMORY.md：⏭️ 跳过（将在第 5 轮提炼）
- ReCAP 重分解：⚠️ 建议下轮重分解（当前层级任务已全部完成）
```
