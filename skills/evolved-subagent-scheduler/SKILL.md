# 子代理智能调度器 Skill

**版本**: 1.0
**创建时间**: 2026-03-15
**创建方式**: evolver-self-evolution (Round 325)
**目的**: 智能调度子代理任务，避免 API Rate Limit 和资源冲突

---

## 🎯 核心问题

### PAT-115: 子代理并发导致 API Rate Limit（高优先级）

**症状**:
```
任务: novel-auto-review
错误: Rate limit reached. Please try again later.
出现次数: 90 次（+22%，6 小时内）
持续时间: > 48 小时
影响: 小说评审流程完全中断
```

**影响**:
- 🔴 **子代理任务大量失败**: 逻辑审核员、人物分析师等全部失败
- 🔴 **API Rate Limit 频繁触发**: 多个子代理同时请求 API
- 🔴 **任务流程中断**: 小说评审完全无法进行
- 🟡 **资源浪费**: 失败的子代理仍消耗资源

**根本原因**:
1. **子代理并发启动** - 多个子代理同时请求 API
2. **缺少调度队列** - 没有全局请求管理
3. **未实现失败重试** - 失败后没有自动重试
4. **缺少速率限制** - 未控制请求频率

## 📋 调度策略

### 1. 串行化调度

```
┌─────────────────────────────────────────────────┐
│           子代理智能调度器                        │
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌──────────────┐                               │
│  │  子代理队列   │                               │
│  │  (FIFO)      │                               │
│  └──────┬───────┘                               │
│         │                                       │
│         ▼                                       │
│  ┌──────────────┐    ┌──────────────┐          │
│  │  速率限制器   │───▶│  串行执行器   │          │
│  │  (1 req/s)   │    │  (1 agent)   │          │
│  └──────────────┘    └──────────────┘          │
│                             │                   │
│                             ▼                   │
│                      ┌──────────────┐          │
│                      │  API 调用     │          │
│                      └──────────────┘          │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 2. 调度优先级

| 优先级 | 子代理类型 | 示例 | 执行策略 |
|--------|-----------|------|---------|
| P0 | 关键任务 | 紧急响应 | 立即执行 |
| P1 | 重要任务 | 逻辑审核员 | 优先执行 |
| P2 | 普通任务 | 人物分析师 | 正常执行 |
| P3 | 后台任务 | 统计分析 | 低优先级 |

## 🚨 调度规则

### 1. 串行化执行

- **规则**: 同时只允许 1 个子代理执行
- **原因**: 避免 API Rate Limit
- **例外**: P0 优先级任务可紧急插入

### 2. 速率限制

- **间隔**: 每次请求间隔至少 1 秒
- **突发**: 允许短时间内 3 次突发请求
- **退避**: Rate Limit 触发后延迟 5 秒

### 3. 失败重试

- **次数**: 最多重试 3 次
- **延迟**: 指数退避（5s → 10s → 20s）
- **放弃**: 3 次失败后标记任务失败

### 4. 超时控制

- **超时**: 单个子代理最长 120 秒
- **处理**: 超时后自动取消并重试
- **记录**: 记录超时原因到日志
