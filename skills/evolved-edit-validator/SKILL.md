## 🎯 核心问题

### PAT-013: 幻影编辑失败 (Phantom Edit Failure)

**发现时间**: 2026-03-17 04:00
**严重程度**: P1
**状态**: 🔴 未解决

**症状**:
- edit 工具尝试编辑存在的文件，但内容不匹配
- 错误信息: "Could not find the exact text"
- 34 次失败记录（6 小时内）

**触发条件**:
1. 文件内容在读取后被其他进程修改
2. 尝试编辑动态生成的内容（时间戳等）
3. 编辑上下文不足，无法唯一匹配
4. 空格/换行符不匹配

**影响**:
- 任务链式失败
- 需要多次重试
- 降低系统可靠性

## 📋 验证策略

### 策略 1: Read-Verify-Edit 三步法

**核心理念**: 编辑前总是先验证

```
┌─────────────────────────────────────────┐
│         Edit 验证流程                    │
├─────────────────────────────────────────┤
│                                         │
│  Step 1: Read 文件                      │
│  └─ 获取最新内容                        │
│                                         │
│  Step 2: Verify 文本存在                │
│  └─ 检查 oldText 是否存在               │
│  └─ 检查是否唯一（不重复）              │
│  └─ 检查上下文是否足够                  │
│                                         │
│  Step 3: Execute Edit                   │
│  └─ 如果验证通过，执行 edit             │
│  └─ 如果验证失败，调整策略              │
│                                         │
└─────────────────────────────────────────┘
```

**实现示例**:

```javascript
// ❌ 错误做法：直接编辑
edit({
  path: "file.md",
  oldText: "## 标题",
  newText: "## 新标题"
})

// ✅ 正确做法：先验证再编辑
const content = read({ path: "file.md" })

// 1. 检查文本是否存在
if (!content.includes("## 标题")) {
  console.log("⚠️ 文本不存在，跳过编辑")
  return
}

// 2. 检查是否唯一
const matches = content.match(/## 标题/g)
if (matches && matches.length > 1) {
  console.log("⚠️ 文本重复，需要更多上下文")
  // 提供更多上下文
  edit({
    path: "file.md",
    oldText: "## 标题\n\n这是第一段内容。",
    newText: "## 新标题\n\n这是第一段内容。"
  })
  return
}

// 3. 执行编辑
edit({
  path: "file.md",
  oldText: "## 标题",
  newText: "## 新标题"
})
```

### 策略 2: 上下文增强

**问题**: 简短文本容易重复

**解决方案**: 自动扩展上下文

```javascript
// ❌ 风险：短文本可能重复
oldText: "## 标题"

// ✅ 安全：包含上下文
oldText: `## 标题

这是标题下的第一段内容。
- 列表项 1
- 列表项 2`
```

**自动扩展算法**:

1. 读取文件内容
2. 定位目标文本
3. 提取前后各 2-3 行
4. 使用扩展后的文本进行编辑

---

### 策略 3: 精确匹配检查

**问题**: 空格/换行符不匹配

**解决方案**: 精确复制原文

```javascript
// ❌ 风险：手动输入可能不精确
oldText: "## 标题\n内容"

// ✅ 安全：从 read 结果精确复制
const content = read({ path: "file.md" })
const exactText = extractExactText(content, "## 标题")
// exactText 包含精确的空格、换行、缩进
```

### 策略 4: 失败恢复机制

**问题**: Edit 失败后如何恢复？

**解决方案**: 多级回退策略

```
┌─────────────────────────────────────────┐
│         Edit 失败恢复策略                │
├─────────────────────────────────────────┤
│                                         │
│  Level 1: 重试（更多上下文）             │
│  └─ 扩展 oldText，增加上下文            │
│  └─ 重新执行 edit                       │
│                                         │
│  Level 2: 分段编辑                      │
│  └─ 将大编辑分解为小编辑                │
│  └─ 逐个执行                            │
│                                         │
│  Level 3: Write 替代                    │
│  └─ Read 整个文件                       │
│  └─ 在内存中修改                        │
│  └─ Write 整个文件                      │
│                                         │
│  Level 4: 放弃并记录                    │
│  └─ 记录失败原因                        │
│  └─ 通知用户                            │
│                                         │
└─────────────────────────────────────────┘
```

```markdown
## 任务：更新 config.json

2. Verify "oldSetting" 存在且唯一
3. Edit with verification
```

### 场景 2: 编辑 Markdown 文件

```markdown
## 任务：更新 README.md

1. Read README.md
2. 定位目标章节
3. 提取包含上下文的 oldText
4. Edit with extended context
5. 如果重复，使用更多上下文
```

### 场景 3: 编辑动态文件

```markdown
## 任务：编辑包含时间戳的文件

1. Read 文件
2. 识别动态部分（时间戳等）
3. 避免编辑动态部分
4. 使用静态部分作为 oldText
5. 如果必须编辑动态部分，考虑 Write 替代
```

## 📝 最佳实践

### DO ✅

1. ✅ **总是先 Read** - 获取最新内容
2. ✅ **验证文本存在** - 避免"幻影编辑"
3. ✅ **提供足够上下文** - 确保唯一性
4. ✅ **精确匹配** - 包括空格和换行
5. ✅ **准备恢复策略** - 失败时自动降级

### DON'T ❌

1. ❌ **不要假设文本存在** - 可能已被修改
2. ❌ **不要使用简短文本** - 容易重复
3. ❌ **不要忽略空白字符** - 必须精确匹配
4. ❌ **不要放弃恢复** - 总是有替代方案
5. ❌ **不要跳过验证** - 节省时间的代价是更多失败
